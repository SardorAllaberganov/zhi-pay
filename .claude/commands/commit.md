# /commit

Group uncommitted changes by topic and create separate commits for each group, then push.

### Step 1: Analyze Changes
Run `git status` and `git diff` (staged + unstaged) to see all modified / untracked files.

### Step 2: Group by Topic
Classify each changed file into a topic group:

**Grouping rules (in priority / dependency order):**
1. **Docs — Models & Schema** — `docs/models.md`
2. **Docs — PRD** — `docs/product_requirements_document.md`
3. **Docs — Flows & State Machines** — `docs/mermaid_schemas/*`
4. **Design Tokens** — `design/tokens/*`
5. **Design Primitives** — `design/primitives/*`
6. **Design Components** — `design/components/*`
7. **Design Patterns** — `design/patterns/*`
8. **Screens (Mobile)** — `design/screens-mobile/*`
9. **Screens (Admin)** — `design/screens-admin/*`
10. **Flows** — `design/flows/*`
11. **Research** — `design/research/*`
12. **Localization (Copy)** — `i18n/*`
13. **Rules** — `.claude/rules/*`
14. **Commands** — `.claude/commands/*`
15. **Orientation** — `ai_context/*`
16. **Orchestration / Config** — `CLAUDE.md`, `.claude/settings.json`, `.claude/settings.local.json`

If files are tightly coupled for one logical change, group them together (e.g. a state-machine update in `docs/mermaid_schemas/` + the rule that references it + the screen that visualizes it).

### Step 3: Present Plan
Show the proposed commit groups:

```
Group 1: [topic] — N files
  - file1.md
  - file2.md
  Commit message: "type(scope): description"

Group 2: [topic] — N files
  - file3.md
  Commit message: "type(scope): description"
```

Wait for user approval before proceeding.

### Step 4: Commit Each Group
For each approved group, in dependency order (docs first → orchestration last):
1. `git add <specific files>`
2. `git commit -m "<message>"`

**Commit message format:**
- Prefix: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`, `test:`, `content:`
  - `content:` is for copy / localization changes (i18n keys, `error_codes` messages, screen text)
- Include scope: `feat(send-money): add rate-lock countdown pattern`
- First line under 72 chars
- Add body if needed for context
- NEVER include "Co-Authored-By", "Generated by", or any AI / Claude signature

### Step 5: Push
```bash
git push
```

### Rules
- NEVER use `git add -A` or `git add .` — always add specific files
- NEVER commit `.env`, `node_modules/`, `logs/`, design tool caches, raw exports, or any binary > 5MB
- Never include AI attribution lines in commit messages
- If unsure about grouping, ask the user
- If there are merge conflicts or hook failures, stop and report
